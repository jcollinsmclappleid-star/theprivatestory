Build the MoonlitAudio backend engine using the following exact prompt system, JSON schemas, save logic, memory logic, and variation logic.

IMPORTANT:
- This spec is for the engine, not just the UI.
- The goal is consistent, high-quality, emotionally deep personalised audio stories.
- The engine must use a hidden planning layer before writing.
- User input is minimal, but output quality must still feel rich and intentional.
- The product is adult in tone, intimate, cinematic, and sensual but NOT explicit pornography.

==================================================
1. ENGINE OVERVIEW
==================================================

Use this exact generation pipeline:

1. User submits minimal intake
2. System normalises intake
3. Hidden Story Planner creates structured story brief
4. Story Writer creates multi-scene story
5. Quality Control evaluates output
6. If weak, targeted rewrite happens
7. Image Prompt Generator creates cover + scene prompts
8. Image Generator creates visuals
9. Audio Generator creates narration
10. Save story to user library
11. Save/update taste profile for recommendations
12. Enable variation / continue-story logic

3. INPUT NORMALISATION LAYER
==================================================

Before planning, normalise raw user input.

Function: normaliseIntake(input)

Raw user input:
{
  "name": "Ava",
  "mood": "Late Night",
  "intensity": "Warm",
  "length": "5 min",
  "scenario": "meeting an old school friend again in a hotel bar",
  "cinematic_visuals": true,
  "emotional_focus": true,
  "voice_feel": "Deep Voice"
}

Normalised output should:
- trim whitespace
- map invalid values to safe defaults
- clamp length to allowed options
- ensure scenario is at least 6–10 meaningful words
- if scenario too short, enrich slightly without changing meaning

Defaults:
- mood: Emotional
- intensity: Warm
- length: 5 min
- cinematic_visuals: true
- emotional_focus: true
- voice_feel: Soft Voice

If scenario is missing:
Fallback scenario:
“an unexpected late evening encounter that becomes emotionally charged”

==================================================

Route:
POST /api/qc-story

This is required.

Use this system prompt exactly:

SYSTEM PROMPT:
You are a quality-control editor for a premium audio storytelling platform.
You do not rewrite the whole story unless necessary.
You evaluate whether the story feels emotionally deep, specific, well-paced, structured, and commercially publishable.

Use this user prompt template exactly:

USER PROMPT:
Evaluate this story against the brief.

Brief:
{{brief_json}}

Story:
{{story_json}}

Score the story from 1 to 10 on:
- emotional_depth
- specificity
- pacing
- scene_progression
- originality
- sensory_detail
- ending_strength

Also return:
- passed (true/false)
- score_total
- issues array
- rewrite_strategy

Rules:
- pass threshold = 7.5 total
- if ending_strength < 7, suggest "rewrite_ending"
- if specificity < 7, suggest "increase_specificity"
- if originality < 6.5, suggest "rotate_dynamic_or_setting"
- if pacing < 7, suggest "tighten_scene_flow"
- if emotional_depth < 7, suggest "increase_vulnerability"

Return JSON only in this format:

{
  "passed": true,
  "score_total": 8.3,
  "sub_scores": {
    "emotional_depth": 8,
    "specificity": 8,
    "pacing": 8,
    "scene_progression": 9,
    "originality": 7,
    "sensory_detail": 9,
    "ending_strength": 8
  },
  "issues": [],
  "rewrite_strategy": null
}
