Build a premium web app called MoonlitAudio.

This is an AI-powered immersive audio storytelling platform with a dark, luxurious, cinematic interface.

IMPORTANT:
- Build from scratch based on this specification.
- Use a modern, clean, premium UI.
- Do NOT make it look like a generic demo.
- The product should feel like Netflix x Calm x Spotify, with tasteful adult romantic animation styling.
- The tone should be intimate, emotionally charged, cinematic, warm, and sensual — but NOT explicit pornography.
- Focus on a premium consumer product, not a developer tool.

==================================================
1. PRODUCT VISION
==================================================

MoonlitAudio is an immersive storytelling platform where users can:

1. Browse a premium library of curated audio stories and series
2. Search by mood, category, tone, or title
3. Click “Create Your Story”
4. Fill out an elegant onboarding intake
5. Generate a personalised story experience using AI
6. Receive:
   - generated title
   - generated description
   - structured story split into scenes
   - narrated audio using ElevenLabs
   - AI-generated cover image
   - AI-generated scene images
7. Play the story while visuals change as playback progresses

The app must feel emotional, addictive, private, immersive, and premium.

==================================================
2. KEY PRODUCT DECISION
==================================================

DO NOT generate one image per minute.

For V1, use:
- 1 cover image
- 1 image per scene
- 4 to 6 scenes per generated story

Images should change based on story progress.

Reason:
- lower cost
- faster load times
- better UX
- still feels immersive

This is an audio-first product with scene-anchor visuals.

==================================================
9. STORY GENERATION (GPT)
==================================================

Use OpenAI to generate a structured story.

The story must be:
- second person (“you”)
- emotionally immersive
- intimate
- cinematic
- sensual but not explicit
- adult in tone
- slow and atmospheric
- not generic
- not over-expository

Return JSON in this exact shape:

{
  "title": "...",
  "description": "...",
  "scenes": [
    {
      "id": 1,
      "text": "...",
      "visual_prompt": "...",
      "duration_estimate": 60
    }
  ]
}

Story requirements:
- 4 to 6 scenes
- each scene should be roughly 45–90 seconds
- each scene must include a visual prompt for image generation
- the story should feel like it is happening to the listener
- strong emotional subtext
- slow build
- good pacing
- premium language
- tasteful adult romance energy

Use this GPT instruction style:

“Write an immersive, second-person audio story.
Make the listener feel like the story is happening to them.
Prioritise atmosphere, pauses, subtext, emotional tension, and anticipation.
Avoid generic narration, clumsy exposition, and explicit sexual detail.
The tone should be intimate, cinematic, emotionally charged, and adult.”

11. AUDIO GENERATION (ELEVENLABS)
==================================================

Combine all scene text into one full narration script.

Use ElevenLabs to generate a single audio file.

Use a warm, intimate delivery style.

Recommended voice settings:
- stability: 0.6
- similarity_boost: 0.7
- style: 0.8

Return:
- playable audio URL or stored audio file path

==================================================
12. SCENE / IMAGE SYNCING
==================================================

As audio plays:
- switch visuals based on playback progress
- use scene boundaries, not per-minute generation

If exact timestamps are difficult:
- approximate by dividing audio progress by number of scenes

Example:
if progress is 0–20% show scene 1 image
20–40% show scene 2
etc.

Use smooth fades between images.
