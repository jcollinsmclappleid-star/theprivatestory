9. QUALITY CONTROL ENGINE
==================================================

This must be a real engine, not just an idea.

After story generation, run a QC evaluation before passing output forward.

Route:
POST /api/qc-story

Input:
{
  brief,
  story
}

Use OpenAI or internal rules.

QC must score the story on:
- emotional_depth
- specificity
- pacing
- scene_progression
- originality
- sensory_detail
- ending_strength

Return:

{
  "passed": true,
  "score_total": 8.4,
  "sub_scores": {
    "emotional_depth": 8,
    "specificity": 9,
    "pacing": 8,
    "scene_progression": 8,
    "originality": 7,
    "sensory_detail": 9,
    "ending_strength": 8
  },
  "issues": []
}

Hard rules:
- if score_total < 7.5 → regenerate
- if ending_strength < 7 → regenerate ending only
- if specificity < 7 → rewrite generic lines
- if originality < 6.5 → force variation prompt and regenerate

Create a rewrite path:
- /api/qc-story can return a corrected prompt for targeted rewrite
- do not always regenerate from zero
- if only one part is weak, fix only that part
