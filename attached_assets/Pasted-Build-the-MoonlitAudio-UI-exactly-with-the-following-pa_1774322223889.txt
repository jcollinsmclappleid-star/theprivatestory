Build the MoonlitAudio UI exactly with the following page structure, hierarchy, components, and behaviors.

IMPORTANT:
- This spec is only for the frontend design / UX structure.
- It should sit on top of the backend generation architecture already defined.
- Keep the interface premium, dark, cinematic, emotionally immersive, and mobile responsive.
- The experience should feel like a luxury streaming product, not a form tool.

==================================================
1. GLOBAL DESIGN SYSTEM
==================================================

A. Visual Theme
- Backgrounds: deep black / charcoal / midnight tones
- Accent colours: plum, burgundy, muted wine, soft gold
- Highlight colour: warm gold for premium emphasis
- Card surfaces: soft translucent dark panels with subtle borders
- Hover glow: plum/gold subtle shadow
- Use gradients sparingly but elegantly

B. Typography
- Headings: serif, elegant, cinematic
- Body text: clean sans-serif, lightly spaced
- Small UI labels: uppercase, letter-spaced, premium feel
- Do not use loud or cheap typography

C. Motion
- Smooth fades
- Soft hover scale
- Image fade transitions
- Gentle panel rise on load
- No aggressive animations

D. Spacing
- Generous white space
- Strong visual hierarchy
- Consistent vertical rhythm

E. Buttons
Primary CTA:
- rounded or softly rounded
- plum / burgundy gradient
- subtle glow
- white text
- hover lift

Secondary CTA:
- dark glass button
- light border
- subtle hover fill

Ghost CTA:
- text button with slight underline or shimmer

==================================================
2. NAVIGATION

Desktop top nav:
- fixed header

- center/right nav links:
  - Home

6. CREATE YOUR STORY PAGE
==================================================

This is the main monetisable feature and must feel special.

----------------------------------
PAGE LAYOUT
----------------------------------

Desktop:
- two-column
- left: intake form
- right: live preview / generated story result panel

Mobile:
- stacked
- form first
- result panel below

----------------------------------

A. Page Header
- eyebrow:
  “Personalised Engine”
- headline:
  “Create a story shaped around you”
- short supporting copy
- premium, emotionally framed

----------------------------------

B. Intake Form Card
Wrap in premium glass/dark card.

Fields:
1. Listener Name
2. Mood / Genre
3. Intensity
4. Voice Feel
5. Story Length
6. Scenario Prompt
7. Optional toggles

Form behavior:
- large inputs
- generous spacing
- premium labels
- placeholder examples
- clear hierarchy

Buttons:
- primary:
  “Create My Story”
- secondary:
  “Use Sample”

Optional helper text:
- “A sentence is enough. We’ll shape the rest.”

----------------------------------

C. Loading State Card
When generation starts:
- show dedicated loading panel
- not just button spinner

Display:
- heading:
  “Creating your story…”
- short line:
  “Building the emotional arc, visuals, and narration.”
- animated bars or waveform
- optional scene-building progress messages:
  - Planning emotional arc
  - Writing scenes
  - Generating visuals
  - Rendering narration

Must feel premium and reassuring.

----------------------------------

D. Generated Result Panel
This should feel like a reveal.

Structure:
- large cover image
- gradient overlay
- mood badge
- generated title
- generated description
- tag pills
- scene image area
- audio player
- action buttons

Actions:
- Play
- Save
- Regenerate Variation
- Continue This Story
- Download audio (optional)
- Create another

Optional hidden area:
- “View generation details”
  - brief summary
  - image style direction
Do not surface technical prompts prominently.

14. VARIATIONS UX
==================================================

When user clicks “Regenerate Variation”, open a premium modal or drawer.

Variation options:
- Softer
- Darker
- Slower
- More Emotional
- New Ending
- New Setting
- Continue the Chemistry

Modal layout:
- title
- short explanation
- option cards
- confirm button

After generation:
- show new version in same result area
- allow compare / replace / save both

==================================================
15. CONTINUE THIS STORY UX
==================================================

When user clicks “Continue This Story”:
- open a confirm modal
- copy:
  “Continue the emotional thread and generate the next chapter.”

Options:
- keep same mood
- raise emotional stakes
- softer continuation
- unresolved continuation

Then create new chapter and save in library as linked story.

17. AMBIENT SOUND UX
==================================================

- city at night
- train
- firelight
- ocean
- quiet room

UI:

- choose ambience
- set ambience volume separately

Default behavior:

18. LOCKED / PREMIUM STATES
==================================================

Design locked content elegantly.

Locked card treatment:
- slightly dimmed image
- lock badge
- subtle premium border

Locked CTA examples:
- Unlock Full Story
- Start Free Trial
- Get More Story Credits

Do not make locked states feel broken.

==================================================
19. MOBILE BEHAVIOR
==================================================

Must be properly designed, not just shrunk.

Mobile priorities:
- strong hero still works
- stacked create flow
- fixed bottom nav
- thumb-friendly buttons
- horizontal content rows remain smooth
- player remains usable
- generated story result feels premium on small screen

==================================================
20. LOADING / EMPTY / ERROR STATES
==================================================

Every state must be designed.

A. Loading
- shimmer cards
- loading bars
- premium copy

B. Empty library
- clean illustration or gradient block
- CTA to create story

C. No search results
- suggest another mood
- CTA to browse or create

D. Generation error
- elegant error card
- retry button
- fallback copy:
  “Something interrupted the story. Try again.”
