Evaluate and provide recommendations and enhancements to analysis and plan # SEO & LLM Infrastructure Audit — The Private Story
April 2026

---

## Executive Summary

The Private Story has a strong SEO foundation: a well-structured llms.txt,
comprehensive schema in index.html, 24 keyword-targeted landing pages, a valid
sitemap, proper robots.txt, and good internal linking.

There is one architecture-level issue that undermines much of it: the site is a
client-side rendered React SPA (Vite). All 24 SEO landing pages, all per-page
title tags, all per-page JSON-LD schemas, and all canonical tags are set by
JavaScript running in the browser — which Googlebot cannot reliably see without
executing that JavaScript first.

Modern Googlebot does render JavaScript via its Web Rendering Service, but this
introduces a render delay of days to weeks, makes results unpredictable, and
gives no guaranteed indexation of any specific page's content.

## What Has Been Implemented

### 1. index.html Foundation — Strong

The static index.html (what every crawler sees immediately, before JavaScript)
contains the following:

- Title: "The Private Story"
- Meta description: 155 characters, homepage-level
- og:type = website
- og:site_name = "The Private Story"
- og:title, og:description set
- og:image = https://theprivatestory.com/opengraph.jpg with 1200x630 dimensions
- og:url (homepage)
- og:locale = en_GB
- twitter:card = summary_large_image
- twitter:title, twitter:description, twitter:image set
- link rel="canonical" pointing to homepage (defaults to homepage for all pages)
- Organization JSON-LD — inline in HTML, not JS — guaranteed crawlable
- WebSite JSON-LD — inline in HTML, not JS — guaranteed crawlable
- WebApplication JSON-LD with pricing offers — inline, guaranteed crawlable
- noscript navigation block linking to all 24 SEO pages — discoverability fallback
- GA4 script (cookie-less consent mode)

The Organization + WebSite + WebApplication @graph block is correctly embedded
as static HTML. This ensures Google's Knowledge Graph receives entity identity
signals on every request.

---

### 2. Per-Page SEO — Present but JavaScript-Dependent

All 24 SEO pages use the useSEO hook and SEOPage component. Before JavaScript
executes, every SEO page returns the same HTML: title "The Private Story",
homepage description, and homepage canonical. If Googlebot indexes before
rendering, it indexes homepage content on every SEO URL — a soft duplicate
content situation.

Elements set via JavaScript (not guaranteed crawlable without JS rendering):
- Page title (document.title in useEffect)
- Meta description (DOM manipulation in useEffect)
- og:title, og:description, og:url (DOM manipulation in useEffect)
- link rel="canonical" (DOM manipulation — defaults to homepage before JS runs)
- FAQPage JSON-LD (document.head.appendChild in useEffect)
- BreadcrumbList JSON-LD (document.head.appendChild in useEffect)
- WebPage JSON-LD (document.head.appendChild in useEffect)
- All H1, H2 headings (React render)
- All body content (React render)

## Recommendations

### CRITICAL — Architecture

Migrate SEO pages from React SPA to Express SSR

The 24 SEO pages should return full, complete HTML from the server — not wait
for React to mount. This makes every page's title, description, canonical, and
all schema blocks guaranteed crawlable. The 24 pages have well-structured
SEOPageConfig data objects that map cleanly to an SSR template pattern.

Impact: Transformative — unlocks all other schema and meta tag improvements.

---

### HIGH IMPACT

1. Fix the canonical tag default
   Before JavaScript runs, every URL on the site sends:
   link rel="canonical" href="https://theprivatestory.com/"
   This signals to Google that all pages are duplicates of the homepage.
   Resolves automatically with SSR migration.

2. Differentiate robots.txt by bot type
   Allow Googlebot, GPTBot, ClaudeBot, PerplexityBot (these send traffic).
   Block CCBot, Common Crawl, and training scrapers (these train future models
   without sending traffic). This is the correct signal: you want to appear in
   ChatGPT and Perplexity answers, but not to train their next model for free.

3. Add HowTo schema to How It Works sections
   The 3-step How It Works on every SEO page is structured for HowTo schema.
   This directly targets Google's rich result carousels and AI Overview answer
   boxes.

4. Add datePublished and dateModified to WebPage schema
   Currently absent from per-page WebPage schema. Required for freshness signals.
   Set datePublished to launch date, dateModified to last content change date.

5. Add AdultContent signals to relevant schemas
   Add isFamilyFriendly: false and contentRating: "Adult Only 18+" to WebPage
   schema. Prevents SafeSearch conflicts and ensures correct content categorisation.

---

### MEDIUM PRIORITY

6. Add Product/Service schema to the Pricing page
   Enables pricing rich results in SERPs.

7. Add ItemList schema to comparison pages
   /audio-stories-vs-audiobooks, /audio-stories-vs-podcasts,
   /alternatives-to-romance-audiobooks
   Targets comparison/decision-stage queries where AI Overviews are common.

   Each follows the existing SEOPageConfig template.

9. Dynamic lastmod in sitemap
   Tie lastmod to actual content change dates, not a static value.

10. Add theme-color meta tag
    meta name="theme-color" content="#c9a227"
    Sets browser chrome colour on mobile. Minor trust/brand signal.

### STRATEGIC / LONG-TERM

11. /learn/ glossary cluster with DefinedTerm schema
    Terms like "slow burn", "dark romance", "enemies to lovers", "audio erotica"

## ChatGPT / Perplexity Specifically

These systems crawl the web to build context. What they prioritise:

1. Clear, direct content early in the page
   The new "What this is / Who it's for / How it works" intro block addresses
   this. BLUF (Bottom Line Up Front) formatting helps AI citation.

2. The llms.txt file
   Perplexity and similar systems increasingly honour this format. The Private
   Story's llms.txt is genuinely well-structured.

3. FAQPage JSON-LD
   Directly used by AI Overviews and Perplexity answer synthesis. The 5-7 FAQ
   entries per SEO page are strong candidates for citation — once guaranteed
   crawlable via SSR migration.

4. Brand entity in Knowledge Graph
   The Organization + WebSite schema in index.html is correctly structured to
   build Google's entity for the brand. This is what makes the site appear
   correctly when someone asks ChatGPT "what is The Private Story?"

5. Specific, verifiable content
   Both GPT and Perplexity prefer to cite pages with specific claims over vague
   marketing prose. The existing SEO pages contain citable specifics: "generated
   by Mistral Large", "narrated by ElevenLabs", "created from seven structured
   choices". This is a genuine competitive strength.
